# System for Cross-domain Identity Management (SCIM)

This page provides an overview of Contentstack SCIM documentation, including how SCIM works, related concepts like group mapping, supported identity providers, setup guides, and reference/FAQ links. It is intended for developers and administrators who need to automate user and group provisioning with an identity provider (IdP).

## System for Cross-domain Identity Management (SCIM)

Contentstack supports SCIM (System for Cross-domain Identity Management), which helps you automate provisioning or deprovisioning of users and groups. It also enables you to update users and groups using an identity provider (IdP).

## Common questions

### What is SCIM used for in Contentstack?
Contentstack supports SCIM (System for Cross-domain Identity Management), which helps you automate provisioning or deprovisioning of users and groups.

### Can SCIM update users and groups through an IdP?
It also enables you to update users and groups using an identity provider (IdP).

### Where can I find setup guides for specific identity providers?
See the **SCIM Guides** section for OneLogin, Microsoft Azure AD, Okta, and Okta Native App links.

### Where is the SCIM API reference?
