# [Contentstack Launch] - Launch Limitations

This page lists the current Launch limitations for Contentstack Launch. It is intended for developers and administrators planning Launch usage, capacity, and project/environment setup, and should be referenced when configuring organizations, projects, environments, domains, deployment hooks, and monthly build hours.

## Launch Limitations

- Number of Launch projects per organization is **10**.
- Number of Launch environments per organization is **30**.
- Number of Launch environments per project is **10**.
- Maximum Launch domains per project is **10**.
- Maximum Launch domains per environment is **2**.
- Maximum deployment hooks per environment is **10**.
- Number of Launch build hours per month is **50**.

## Common questions

### Where can I find the limits for projects and environments in Launch?
On this page under **Launch Limitations**, which lists limits for projects per organization and environments per organization/project.

### How many domains can I add in Launch?
The page lists **Maximum Launch domains per project is 10** and **Maximum Launch domains per environment is 2**.

### Is there a limit on deployment hooks?
Yes, **Maximum deployment hooks per environment is 10**.

### How many build hours are included per month?
The page lists **Number of Launch build hours per month is 50**.
