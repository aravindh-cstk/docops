# [Author Content] - Remove Asset Version Names

This page explains how to remove a custom name from an asset version in Contentstack so it reverts to the default version naming convention. It is intended for content managers and authors who manage assets and need to reset version names during asset maintenance or cleanup.

## Remove Asset Version Names

Asset versions in Contentstack can have custom names for better organization and clarity. You can remove a custom name and revert the version to its default naming convention (e.g., "Version 2").

To remove an [asset](/docs/content-managers/author-content/about-assets) version name, log in to your [Contentstack account](https://www.contentstack.com/login) and follow these steps:
- Go to your [stack](/docs/developers/set-up-stack/about-stack) and click the “Assets” icon in the left navigation panel.
- Navigate to the asset you want to update.
- Click the **version** **dropdown** in the top-right corner of the page.
- From the dropdown menu, view the list of all asset versions. Click the **Rename** icon next to the version you want to name.
- In the input field, remove the custom name (or leave it blank).
- Click the **checkmark** icon or press **Enter** to save the changes.

The version name will reset to its default value (e.g., "Version 2").

**Note:** Removing a version name does not affect the version's content.

## Common questions

### Does removing a version name delete or change the asset version?
No. Removing a version name does not affect the version's content.

### What will the version be called after I remove the custom name?
The version name will reset to its default value (e.g., "Version 2").

### Where do I find the option to remove a version name?
Open the asset, click the **version** **dropdown** in the top-right corner, then use the **Rename** icon for the version and remove the custom name in the input field.

### Do I need to type anything to reset the name?
No. In the input field, remove the custom name (or leave it blank), then save.
