# [Author Content] - Bulk Unpublish Assets

This page explains how content managers and authors can use Contentstack’s Bulk Unpublish Assets feature to unpublish multiple assets at once by selecting assets, choosing environments and languages, and completing the unpublish action.

## Bulk Unpublish Assets

Contentstack’s Bulk Unpublish Assets feature helps to unpublish multiple assets. It covers the process of selecting assets, specifying environments and languages, and scheduling unpublishing actions.

To bulk unpublish assets, log in to your [Contentstack account](https://www.contentstack.com/login), and perform the following steps:
- Go to your [stack](/docs/developers/set-up-stack/about-stack) and click the “Assets” icon in the left navigation panel. You can also use the shortcut key “A” (for both Windows and Mac OS).
- Use the checkboxes to select the assets you want to unpublish.
- After selecting the assets, click the **Unpublish** option in the floating panel that appears.
- In the **Unpublish** **Assets** modal:**Select Environment(s):** Select the environment(s) from which the assets must be unpublished.
- **Select Language(s):** Select the locale(s) in which you want the assets to be unpublished. If a localized asset exists, it will be selected automatically.
- Click **Unpublish** to unpublish the selected assets.

**Note: **You can unpublish up to **10** **assets** simultaneously in **10** **languages** across **10** **environments**.

## Common questions

### Can I unpublish assets from multiple environments at once?
Yes. In the **Unpublish** **Assets** modal, use **Select Environment(s):** to choose the environment(s) from which the assets must be unpublished.

### Can I unpublish assets in multiple languages at once?
Yes. Use **Select Language(s):** to select the locale(s) in which you want the assets to be unpublished.

### What happens if a localized asset exists?
If a localized asset exists, it will be selected automatically.

### Are there limits to bulk unpublishing?
Yes. You can unpublish up to **10** **assets** simultaneously in **10** **languages** across **10** **environments**.
