# [Author Content] - Name Asset Versions

This page explains how to assign custom names to asset versions in Contentstack to help content managers and authors organize and identify specific asset versions. Use it when you need to rename an asset version (instead of relying on default version numbers) for clearer content management workflows.

## Name Asset Versions

Assigning custom names to versions can help you better organize and identify specific versions of your assets for improved content management.

By default, Contentstack assigns a version number to each asset (e.g., "Version 1," "Version 2," etc.) . To make versions easier to identify, you can assign custom names, such as "Initial Image for Banner" or "Do Not Edit."

To assign a name to an [asset](/docs/content-managers/author-content/about-assets) version, log in to your [Contentstack account](https://www.contentstack.com/login), and perform the following steps:

- Go to your [stack](/docs/developers/set-up-stack/about-stack) and click the “Assets” icon in the left navigation panel.
- Navigate to the asset you want to modify.
- Click the **version** **dropdown **located in the top-right corner of the page.
- From the dropdown menu, view the list of all asset versions. Click the **Rename** icon next to the asset version you want to rename.
- Type the new name for the version in the provided field. Once done, confirm your changes by either clicking the **checkmark** icon or pressing **Enter** on your keyboard.

**Note**: Custom version names have a maximum limit of **32** **characters**.

## Common questions

### Can I rename any asset version or only the latest one?
You can rename an asset version by selecting it from the version dropdown and clicking the **Rename** icon next to the version you want to rename.

### What happens if I exceed the custom version name limit?
Custom version names have a maximum limit of **32** **characters**.

### Do custom version names replace the default version numbering?
Custom names help identify versions more easily, while Contentstack assigns a version number to each asset by default (e.g., "Version 1," "Version 2," etc.).
