# [Create and Manage Releases NEW] - Releases Limitations

This page lists the limitations and constraints for Releases, including field length limits, API item limits per request, and behavior notes for updating release items. It is intended for content managers and developers who create or manage releases and need to understand operational boundaries and update behavior.

## Releases Limitations

- Max character length of the ‘Title’ field is **50**.
- Max items allowed to be added in a Release, in a single instance via API, is **100**.
- The [Update release items to their latest versions](http://docs/content-managers/create-and-manage-releases/update-release-items-to-their-latest-versions) option doesn't add the new references added to an updated version of a release item automatically. You need to add them manually.

## Common questions

**Q: What is the maximum character length allowed for a Release title?**  
A: Max character length of the ‘Title’ field is **50**.

**Q: How many items can be added to a Release via API in a single instance?**  
A: Max items allowed to be added in a Release, in a single instance via API, is **100**.

**Q: Does updating release items to their latest versions automatically include new references?**  
A: No. The option doesn't add the new references added to an updated version of a release item automatically. You need to add them manually.
