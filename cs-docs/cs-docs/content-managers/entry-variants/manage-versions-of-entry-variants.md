# [Variants CMS] - Manage Versions of Entry Variants

This page explains how to manage versions of entry variants in Contentstack, including how to view, compare, and restore versions. It is intended for content managers and editors who work with Entry Variants and need to track changes over time or revert to earlier versions.

## Manage Versions of Entry Variants

**Note**: The Entry Variants feature is currently available as part of an Early Access Program and may not be available to all users. For more information, you can reach out to our [support](mailto:support@contentstack.com) team.

Managing versions of entry variants allows you to keep track of changes made over time, restore previous versions, and maintain a clear history of content updates.

Every time you update an entry variant, a new version of that entry variant is saved. You can view and manage these versions to keep track of changes, compare them, or restore a previous version if needed.

To manage the versions of an entry variant, log in to your [Contentstack account](https://www.contentstack.com/login) and perform the steps given below:
- Navigate to the desired [stack](../../developers/set-up-stack/about-stack.md), then click the **Entries **icon in the left navigation panel.
- Select an existing entry variant for which you want to manage versions.

Now that you have accessed the relevant entry variant, you can easily view, compare, and restore versions as needed.

## View Versions
- To view the versions of an entry variant, click the “Versions” dropdown located in the top-right corner of the entry editor.
- A list of all versions of the entry variant, with their version numbers, will be displayed.
- Click any version from the list to view its details.

## Compare Versions
- To compare two different versions, hover over the version you want to compare, and then click the “Compare” icon.
- All the differences between the selected versions will be highlighted.

## Restore a Version
- If you need to revert to a previous version, select the version from the list that you want to restore.
- Click the **Save **button to save it as the latest version.

## Best Practices for Managing Versions of Entry Variants
- **Regular Review:** Regularly review versions to keep track of changes.
- **Naming Conventions:** Use meaningful names or descriptions when saving new versions to easily identify their purpose or updates.
- **Limit Versions:** Ensure all necessary updates are made before saving to minimize the creation of excessive versions.
- **Collaborate Wisely:** Communicate with your team members about version changes to prevent accidental overwrites or loss of critical information.

By following these steps, you can effectively manage entry variant versions in Contentstack, ensuring content accuracy, clear history tracking, and the ability to revert changes when necessary.
