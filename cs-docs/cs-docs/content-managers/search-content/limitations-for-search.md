# [Search Content] - Limitations for Search

This page lists the limitations for Search Content. It is intended for content managers who need to understand search term, saved view, publishing, and workflow constraints when working with search results.

## Limitations for Search

- The maximum size of a search term (in bytes) is **32,766**.
- You can save a maximum of **500**** views, **including both entries and assets.
- At a time, you can publish **10** **entries** in **10** **languages** and in **10** **environments** from the search results.
- You can add up to **100** **users** and **100** **roles** for each stage while updating the workflow details.
- You can change the workflow stage of multiple entries only if all the entries have been assigned the same workflow stage and are associated with the same workflow.
- You can change the workflow stage of all the selected entries to the same new stage at once. For example, you can move entries that are currently in the “Ready for Review” stage to the “Complete” stage in one action.
