This page lists the Entry Versions documentation section and its related tasks. It is intended for content managers and developers who need to manage, name, rename, remove names, compare, restore, or understand entry versions.

### Introduction

By default entry versions are identified with numbers (for e.g., Version 1, Version 2). However, you can assign a name to a version for easy identification (for e.g., Production Ready, Do Not Edit).

## Common questions

### What are entry versions identified with by default?
By default entry versions are identified with numbers (for e.g., Version 1, Version 2).

### Why would I assign a name to a version?
You can assign a name to a version for easy identification (for e.g., Production Ready, Do Not Edit).
