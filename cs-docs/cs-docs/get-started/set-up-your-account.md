# [Get Started with CS] Set Up your Account

This page explains the key tasks an organization owner should complete when creating a new Contentstack account, including selecting a region, securing access, inviting teammates, migrating content, and finding support and community resources.

Learn how to set up your organization, data center location, migration or import of content, account security, and other attributes while creating a new [Contentstack account](https://app.contentstack.com/#!/login).

The following outlines the account setup tasks you should complete as an organization owner. Links to detailed documentation are given within each section.

## Select a Region for your Datacenter
A Contentstack region refers to the location of the data centers where your organization's data resides. Contentstack currently supports seven regions: AWS North America, AWS Europe, AWS Australia, Azure North America, Azure Europe, GCP North America, and GCP Europe. While subscribing to a Contentstack account, you can decide the region that will host your organization’s data.

Learn more about [Regions](/docs/developers/contentstack-regions/about-regions) and [Login Endpoints](/docs/developers/contentstack-regions/login-endpoints).

## Secure your Account
Sign up for your [Contentstack account](https://app.contentstack.com/#!/login) to start with your digital content management experience. The standard process of signing up is with your name and email address. If your organization has an SSO set up, you can also sign up via SSO. To improve account security and reduce the chance of unauthorized access, you can set up [two-factor authentication](/docs/developers/security/two-factor-authentication/) (2FA) or [single sign-on](/docs/developers/single-sign-on/) (SSO).

Learn more about [Password Requirements](/docs/developers/security/password-requirements/) and [Account Lockout Policy](/docs/developers/security/account-lockout-policy/).

## Invite your Team
Start collaborating and working in a stack by inviting your team members. In Contentstack, "users" refers to all member accounts in a stack. Instead of granting access to each user individually, Contentstack makes use of roles to grant permissions to a group of users.

Learn more about [Users](/docs/developers/invite-users-and-assign-roles/about-stack-users/) and how to [Assign Role to a User](/docs/developers/invite-users-and-assign-roles/assign-role-to-a-user).

## Migrate your Content
Content migration is the process of transferring content from one CMS to another. You must first export the content from your current CMS and then import it into Contentstack.

Learn more about [Export Guides for CMSs](/docs/developers/content-migration/#export-guides-for-cmss) and how to [Import Content into Contentstack](/docs/developers/import-content-into-contentstack/).

## Common questions

### Who should complete these setup tasks?
These tasks are intended for an organization owner setting up a new Contentstack account.

### When do I choose my organization’s region?
While subscribing to a Contentstack account, you can decide the region that will host your organization’s data.

### What are recommended options to improve account security?
You can set up [two-factor authentication](/docs/developers/security/two-factor-authentication/) (2FA) or [single sign-on](/docs/developers/single-sign-on/) (SSO).

### Where can I get help if I run into issues?
Contact [support](mailto:support@contentstack.com) and review the resources for your [support](/docs/faqs/#support-faqs).
