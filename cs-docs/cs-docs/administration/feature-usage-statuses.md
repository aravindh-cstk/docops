# [Platform Discovery] - Understand Feature Usage Statuses

This page explains how Platform Discovery assigns feature usage statuses across your organization, who should use these statuses to assess adoption and optimization opportunities, and when to reference them while reviewing feature cards and platform usage trends.

## Understand Feature Usage Statuses

Platform Discovery evaluates feature usage across your organization and assigns a status to each feature card.

These statuses help you understand adoption trends, identify underused capabilities, and evaluate opportunities for optimization.

## Active

In Platform Discovery, active features display a green status indicator representing platform usage.

Depending on the feature, activity may include:
- User interactions
- Workflow executions
- API usage
- Deployments
- Content updates
- Feature configuration

Most activity checks evaluate usage within the last **90 days**.

## No Recent Activity

Features with no recent activity display a red status indicator representing potential adoption opportunities.

This status can help identify:
- Features that may need additional onboarding.
- Features that are configured but unused.
- Opportunities to improve platform adoption.

## Requires Plan Upgrade

A feature is marked as **Requires Plan Upgrade** when the capability is unavailable in your current subscription plan.

These features display a yellow status indicator representing expansion opportunities.

These features remain visible to help you:
- Explore additional platform capabilities.
- Understand advanced feature availability.
- Evaluate future expansion opportunities.

**Tip**: Contact Contentstack support to learn more about enabling upgraded features.

## Learn More About a Feature

Each feature card includes a **Learn More** option.

Use this option to:
- Access product documentation.
- Review implementation guidance.
- Explore Academy videos.
- Understand common use cases.

## Common questions

### What does the Active status mean in Platform Discovery?
Active features display a green status indicator representing platform usage, and most activity checks evaluate usage within the last **90 days**.

### What does No Recent Activity indicate?
Features with no recent activity display a red status indicator representing potential adoption opportunities, such as features that may need additional onboarding or are configured but unused.

### Why is a feature marked Requires Plan Upgrade?
A feature is marked as **Requires Plan Upgrade** when the capability is unavailable in your current subscription plan, and these features display a yellow status indicator.

### Where can I find more information about a specific feature?
Each feature card includes a **Learn More** option to access product documentation, review implementation guidance, explore Academy videos, and understand common use cases.
