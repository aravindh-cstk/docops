# [Marketplace guides and apps] - App Requests

This page explains how users can request Marketplace app installations when they don’t have install access, and how stack or organization admins can review, approve, install, or reject those requests.

## App Requests

A user who does not have access to install apps from the Marketplace can request the stack or organization admin to get the app installed.  
This can be done by clicking the “Request Install” button on the app information page.

For stack apps, the requesting user can also select the stack for which the app needs to be installed.

## Approving or Rejecting App Requests

Once an App request has been made, the stack or organization admins can view the requests under **Manage** > **App Requests**.

If they approve the request, they can proceed with installing the app, or they can simply reject the request.

To do so, they need to hover on the app request, and click on the respective icon for **Reject Request** or **Install App**.

## Common questions

### Who can request an app installation?
A user who does not have access to install apps from the Marketplace can request the stack or organization admin to get the app installed.

### Where do admins view app requests?
Stack or organization admins can view the requests under **Manage** > **App Requests**.

### What options do admins have after receiving a request?
They can proceed with installing the app, or they can simply reject the request.

### Can a requester choose a stack for stack apps?
For stack apps, the requesting user can also select the stack for which the app needs to be installed.
