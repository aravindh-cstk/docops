# [Set Up Your Project] - Request to Add a New Language to Contentstack

This page explains how to request a new language to be added to Contentstack’s supported languages list, including what details you’ll need to provide and how language codes are determined. It is intended for users managing multilingual stacks who need a language that is not currently available.

## Request to Add a New Language to Contentstack

**Note: **You don’t have to reach out to our support team any more to get a language added to our list of supported languages. You can [add custom languages](./add-a-custom-language.md) of your choice to your stack in a click.

Contentstack supports [over 200 languages](./list-of-supported-languages.md). However, if you don’t find the [language](./about-languages.md) of your choice, you can request us to add it. Here’s how:

## Request for New Language

Contact our support team via the in-product chatbox or [email](mailto:support@contentstack.com) us to request for a new language.

## Provide Language Details

Our support team will ask you to provide the language and the country in which it is being used. Contentstack supports [country-specific languages](./supported-languages.md#country-specific-languages) as well as [generic languages](./supported-languages.md#generic-languages).

## Contentstack Adds the New Language

Contentstack's support team forwards the request to add a new language to the engineering team. The engineering team uses [ISO 639-2 Alpha-3 standard](http://www.loc.gov/standards/iso639-2/php/English_list.php) and  [ISO 3166-1 Alpha-2 country code](https://datahub.io/core/country-list) to get the corresponding language and country designators for the new language.

For country-specific languages, the [language code](./supported-languages.md#language-code-format) comprises two designators (language and country), separated by a hyphen (-). For generic languages, the code contains just one designator (language).

The engineering team will [add the new language](./add-a-language.md) to Contentstack's list of supported languages once the corresponding language code is verified and ready for use.

## Test and Verify the New Language

Once the new language has been added, we will notify you about it. You can add the new language to your stack and try localizing one of your entries in that specific language.

**Additional Resource**: Refer our [Add a Language guide](./add-a-language.md) to learn how to add a language to your [stack](../set-up-stack/about-stack.md).

For any queries please get in touch with our [support team](mailto:support@contentstack.com).

## Common questions

### Can I add a language without contacting support?

Yes. You don’t have to reach out to our support team any more to get a language added to our list of supported languages. You can [add custom languages](./add-a-custom-language.md) of your choice to your stack in a click.

### What information do I need to provide when requesting a new language?

You need to provide the language and the country in which it is being used.

### What standards are used to create the language code?

The engineering team uses [ISO 639-2 Alpha-3 standard](http://www.loc.gov/standards/iso639-2/php/English_list.php) and  [ISO 3166-1 Alpha-2 country code](https://datahub.io/core/country-list).

### How do I verify the new language after it’s added?

Once the new language has been added, we will notify you about it. You can add the new language to your stack and try localizing one of your entries in that specific language.
