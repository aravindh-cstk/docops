# [Developer Hub guides] - Configuring an App

This page explains how to configure an app after installation in Developer Hub, who should read it (developers installing apps), and when to use it (immediately after installing an app and before using it).

## Configuring an App

As soon as you install an app, you are redirected to the **App Configuration **screen. Without providing configuration details, you might not be able to use it.

Lets see an example of configuring an Algolia app.

Here, you need to provide the configuration details of your Algolia account such as the **Application ID**,** Index Name**, and the **API Key **to let your app fetch details to and from Algolia.

These details are configured when you set up your respective app configurations.

Once done, you can choose one of the following actions:
- Simply **Save **the configuration details
- Click **Open Stack **to directly navigate to the stack.
- Click the “More Options” icon (three ellipses) and select **Uninstall the App** to uninstall your app from the stack.

## Common questions

### Do I have to configure an app immediately after installing it?
Yes—after you install an app, you are redirected to the **App Configuration **screen, and without providing configuration details, you might not be able to use it.

### What configuration details might an app require?
For example, an Algolia app requires details such as the **Application ID**,** Index Name**, and the **API Key **.

### What can I do after entering configuration details?
You can **Save **the configuration details, click **Open Stack **, or use “More Options” (three ellipses) to select **Uninstall the App**.

### How do I uninstall an app from the stack?
Click the “More Options” icon (three ellipses) and select **Uninstall the App**.
