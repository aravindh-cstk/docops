# Set Up Your Project - Title

This page explains the default **Title** field available in all content types, what it supports, which properties you can configure, and where to find related documentation. It is intended for developers and content managers configuring content types and working with entries.

### Item 1

#### Article section

##### Heading

Title

##### Content

By default, the Title field is available for all [content types](./about-content-types.md) created. It accepts single-line text and does not provide any formatting options or line breaks. The value entered in this field, while creating an entry, is considered as the title of the [entry](../../content-managers/author-content/about-entries.md).

This field possesses certain [properties](./about-field-properties.md) that you can change at any time as per your needs. The properties that can be modified are “[Display Name](./display-name.md),” “[Placeholder Value](./placeholder-value.md),” “[Instruction Value](./instruction-value.md),” “[Help Text](./help-text.md),” “[Number of Characters](./number-of-characters.md),” “[Validation (Regex)](./validation-regex.md),” and “[Validation Error Message](./validation-error-message.md),” and “[Non-localizable](./non-localizable.md).”

**Additional Resource:** You can read more about [Field Visibility Rules](./about-field-visibility-rules.md) in our documentation where we have covered it extensively.

## Common questions

### Can I remove the Title field from a content type?
By default, the Title field is available for all content types created.

### Does the Title field support multiple lines or formatting?
It accepts single-line text and does not provide any formatting options or line breaks.

### What can I customize for the Title field?
The properties that can be modified are “[Display Name](./display-name.md),” “[Placeholder Value](./placeholder-value.md),” “[Instruction Value](./instruction-value.md),” “[Help Text](./help-text.md),” “[Number of Characters](./number-of-characters.md),” “[Validation (Regex)](./validation-regex.md),” and “[Validation Error Message](./validation-error-message.md),” and “[Non-localizable](./non-localizable.md).”

### Where can I learn more about related rules and validation?
You can read more about [Field Visibility Rules](./about-field-visibility-rules.md) and use [Validation (Regex)](./validation-regex.md) with the title field.
