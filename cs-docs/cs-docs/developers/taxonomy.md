# [Second level navigation] - Taxonomy

This page introduces the Taxonomy section and provides links to related guides for developers who need to classify, organize, and manage content using taxonomy features and term management workflows.

## Listing page

### Heading

Taxonomy

### Introduction

Taxonomy, offers versatile options for classifying, organizing, and managing content according to your unique requirements. The classification methods include hierarchical structuring, relational organization (which may or may not be hierarchical), and predefined lists based on metadata/attributes applicable across diverse use cases.

This feature is intricately designed to streamline the categorization and management of entries, promising a seamless and efficient content organization experience.

## Common questions

### What is this page used for?
It provides the Taxonomy introduction and a structured set of links to Taxonomy and Term workflows, use-cases, and related topics.

### Where do I start if I’m new to Taxonomy?
Use the “About Taxonomy” link, then follow “Create a Taxonomy” and “Create a Term” to begin setting up and using taxonomy.

### What do the NEW flags indicate?
They mark newly added documentation items, with the associated date shown in the “Flag date” column.

### Where can I find limitations or FAQs?
Use the “Taxonomy Limitations” and “Taxonomy FAQs” links in the “More about Taxonomy” section.
