# [Second level navigation] - Single Sign-On (SSO)

This page lists Single Sign-On (SSO) documentation and guides for Contentstack. It is intended for developers and administrators who need to understand how SSO works, configure SSO in Contentstack, or follow provider-specific setup guides.

##### Introduction

Contentstack supports Single Sign-On (SSO). If your Contentstack organization is SSO-enabled, users can access the organization through your corporate identity provider credentials, instead of Contentstack account credentials.

## Common questions

### What is this page used for?
This page is a listing page that groups links to SSO concepts, configuration topics, FAQs, and identity-provider-specific setup guides.

### Where do I start if I’m new to Contentstack SSO?
Use the links under “How SSO Works,” starting with “About Single Sign-On (SSO)” and “How SSO works with Contentstack.”

### Where can I find provider-specific setup steps?
Use the “SSO Guides” section to select your identity provider (for example, Okta, OneLogin, Microsoft Azure AD, or Google G-Suite).

### Where can I find answers to common SSO questions?
Use the “Common Questions on SSO” section links, including “General SSO FAQs” and “Role Mapping FAQs.”
