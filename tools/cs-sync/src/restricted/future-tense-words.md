# Future Tense Words — Restricted List

Technical documentation must use present tense. Future tense misleads users about when things happen.

To add a new word or phrase, add a bullet (`- phrase`) under the appropriate section. No code change is required — the linter reads this file at runtime.

**Bad → Good examples:**
- "The system will display a message" → "The system displays a message"
- "This will allow you to configure" → "This allows you to configure"
- "The page will redirect to" → "The page redirects to"
- "You will see a confirmation" → "You see a confirmation"
- "It shall be saved automatically" → "It is saved automatically"

## Modal Auxiliaries (future-implying)

- will
- shall
